Javascript is called a client-side scripting language. It runs inside an internet browser. Every browser has a Javascript Engine that runs the Javascript code. Such as, Google Chrome has a V8 engine, Firefox has spidermonkey, edge has chakra. This type of Javascript engine at first takes the Javascript source code. Then compile it to the binary instructions. The interpreter takes this bytecode and converts it to the machine code that a CPU can understand. In this way Javascript code is rendered by the browser and we see the final output.

Actually, the event loop is the secret behind Javascript's asynchronous programming. Javascript usually executes its all operation on a single thread. But sometimes we get the illusion of multi-treading. We should know about the call stack term. Cause mainly this is responsible for keeping track of line by line execution of every operation. After finishing a function, it is popped from the stack. Then the event queue is responsible for sending new functions for processing. The set-timeout method is an example of this. The event loop facilitates this process. It constantly checks, if the call stack is empty then new functions are added from the event queue. In this way, we get a cyclic system in Javascript.
